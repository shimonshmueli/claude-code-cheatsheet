Install & Launch

# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | sh

# Or via npm (any OS)
npm install -g @anthropic-ai/claude-code

# Verify
claude --version

# Launch interactive session
claude

# Launch with initial prompt
claude "summarize this project"

# Headless / scripting mode
claude -p "explain this function"

# Continue last session
claude --continue

# Switch model at launch
claude --model sonnet
claude --model haiku
claude --model opus

Key Concepts

SessionOne run of claude from launch to exit. Each session has its own conversation history. Project memory (CLAUDE.md) persists across sessions.
Context windowThe total text — your prompts, Claude's replies, files read — the model holds at once. Fills up over time; managed with /clear and /compact.
TokenUnit for measuring text (~¾ of a word). Usage limits and API billing are counted in tokens. Monitor with /cost.
CLAUDE.mdMarkdown file Claude reads at session start. Holds project conventions, commands, constraints. Lives in project root, home dir, or subfolder. Create with /init.
Plan ModeRead-only mode: Claude explores and proposes but will not edit files or run commands. Toggle: Shift+Tab or /plan.
Accept EditsAuto-approves file edits for the rest of the session; shell commands still prompt. Toggle: Shift+Tab.
MCPModel Context Protocol — open standard for plugging external systems (GitHub, Jira, databases) into Claude as tools. Managed via /mcp.
HookShell command that runs automatically at lifecycle events (before a tool, after an edit). Used for auto-formatting, linting, or blocking unsafe commands.
SkillPackaged instructions + helper files that teach Claude a specific workflow (generate PDF, scaffold component, deploy). Invocable like slash commands; can also auto-activate.

Pipe & Headless

# Pipe output into Claude
git log --oneline | claude -p "summarize"
cat error.log | claude -p "find root cause"

# JSON output for scripting
claude -p "analyze" --output-format json

# Limit turns (cheap CI checks)
claude -p --max-turns 3 "lint check"

# Budget cap (prevent runaway cost)
claude -p "fix tests" --max-budget-usd 2.00

Permission Modes

Default
Claude asks before every file edit and shell command. Safest for unfamiliar codebases.
cycle: Shift+Tab
Accept Edits
File edits auto-approved; shell commands still prompt. Use once direction is trusted.
cycle: Shift+Tab
Plan Mode  read-only
Claude can explore files and reason about the problem but cannot edit files or run commands. Review the plan, then exit Plan Mode to execute. Enter with /plan or Shift+Tab.

CLAUDE.md — Project Memory

Claude reads this file automatically at the start of every session. Survives /compact. The single most powerful way to make Claude feel like it already knows your project.

# Locations (all are read if present)
~/.claude/CLAUDE.md   # personal global
./CLAUDE.md         # project root
./src/CLAUDE.md     # subfolder override

Entry-Level Tips

01Before starting a big task, use /plan to review Claude's approach. Exit Plan Mode to execute only when satisfied — this prevents runaway edits.
02Run /init in every new project. A well-written CLAUDE.md is worth more than a long prompt — it persists across sessions and survives context compaction.
03When the context feels stale or confused, use /clear to reset history while keeping project memory, or /compact to summarize and free space without losing the thread.
04Use @file.py to reference files rather than pasting code. Claude tokenizes file-loaded content more efficiently and paths appear in the audit log.
05Use /rewind if Claude goes off-track mid-task. It rolls back the conversation (and optionally the code state) to an earlier checkpoint — the undo button for agentic work.
06Run /cost after longer sessions to build intuition for token consumption. Use Haiku for simple queries and linting; save Sonnet/Opus for complex reasoning tasks.
07Add CLAUDE_CODE_NO_FLICKER=1 to your shell profile for smoother terminal rendering — output streams line-by-line without screen redraws.
08Pipe output into Claude for quick analysis: cat error.log | claude -p "find root cause". The -p flag runs in headless mode and exits automatically.
09If Claude seems stuck or unhealthy, run /doctor to diagnose the installation, and /bug to report issues with session context attached automatically.
